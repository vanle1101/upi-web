## Local Setup (Recommended)

### Prerequisites

- **Python:** 3.11+ (3.13 preferred)
- **OS:** macOS, Linux, Windows (via WSL2)
- **Internet:** For initial deps + Camoufox Firefox download (~100MB)
- **Disk:** ~2GB free (Python venv + Camoufox binary + SQLite DB)

### Step 1: Clone Repository

```bash
git clone <repo-url> gpt_signup_hybrid
cd gpt_signup_hybrid
```

### Step 2: Run Setup Script

**macOS / Linux:**
```bash
bash setup.sh
```

**Windows (PowerShell):**
```powershell
.\setup.bat
```

The script will:
1. Create Python virtual environment (`.venv/`)
2. Install dependencies from `requirements.txt`
3. Download Camoufox Firefox binary (auto)
4. Create `.env` from `.env.example` (interactive prompts)
5. Initialize SQLite database (`runtime/data.db`)
6. Print connection URL

### Step 3: Start Web UI

```bash
# macOS / Linux
.venv/bin/python -m gpt_signup_hybrid web --host 127.0.0.1 --port 8083

# Windows
.venv\Scripts\python -m gpt_signup_hybrid web --host 127.0.0.1 --port 8083
```

**Output:**
```
INFO:     Uvicorn running on http://127.0.0.1:8083/
INFO:     Press CTRL+C to quit
```

Open browser: `http://127.0.0.1:8083/`

### Step 4: Use Web UI

1. **Settings tab:** Configure proxy pool (optional), mail provider, browser options
2. **Registration tab:** Add combo(s), start jobs, monitor progress
3. **Get Session tab:** Extract session tokens for existing accounts
4. **UPI QR tab:** Generate payment QRs
5. **iCloud HME tab:** Manage Hide-My-Email pool (if enabled)

## Docker Deployment

Production deploy trong container (Camoufox headless qua Xvfb). Artifacts: `Dockerfile`, `docker-compose.yml`, `docker-entrypoint.sh`, `.dockerignore`, `.env.docker.example`.

### Prerequisites

- Docker Engine + Docker Compose v2
- **Arch:** image phải khớp host. Build trên Apple Silicon ra **arm64** — KHÔNG chạy trên VPS amd64. Deploy amd64: build trực tiếp trên VPS, hoặc `docker buildx build --platform linux/amd64 -t gsh:latest .`

### Quick Start

```bash
cp .env.docker.example .env          # set GPT_SIGNUP_WEB_TOKEN (openssl rand -hex 32)
docker compose up -d                 # build (lần đầu) + start web
docker compose ps                    # đợi STATUS = healthy (~40s)
# Mở UI (token KHÔNG auto-embed khi bind container):
#   http://127.0.0.1:8083/?token=<GPT_SIGNUP_WEB_TOKEN>
```

Thiếu `GPT_SIGNUP_WEB_TOKEN` → `compose up` fail-fast với message rõ. `.env` KHÔNG bake vào image (loại trong `.dockerignore`); compose inject qua `env_file`.

### iCloud HME Runner (opt-in)

```bash
docker compose --profile hme up -d   # thêm icloud-hme (chỉ start SAU web healthy)
docker compose logs -f icloud-hme    # xem HmeRunner loop (cycle → wait retry_interval)
```

Mặc định hme TẮT (1 DB writer). `ICLOUD_API_AUTH_TOKEN` chỉ cần nếu dùng iCloud controls trên web UI (không bắt buộc cho CLI runner).

### Persistence & Operations

- `runtime/` (data.db + sessions) ở named volume `gsh-runtime` → survive `docker compose down`.
- `docker compose down` giữ volume; `docker compose down -v` XÓA volume (mất data).
- Migrate chạy tự động lúc start (entrypoint, idempotent).
- Backup: `docker run --rm -v gpt_signup_hybrid_gsh-runtime:/d -v "$PWD":/b alpine tar czf /b/runtime-backup.tgz -C /d .`

### Lưu ý vận hành

- Bind `127.0.0.1:8083` (loopback host) → deploy **sau reverse-proxy + TLS** (nginx/caddy). KHÔNG expose `0.0.0.0` ra internet (`--unsafe-expose-network` chỉ để bind trong container; bảo vệ bằng token + loopback publish).
- Plain compose **KHÔNG auto-restart** container `unhealthy` (chỉ restart khi exit) — dựa reverse-proxy/orchestrator giám sát health.
- `shm_size: 2gb` cho Firefox; `mem_limit: 4g`/service (tinh chỉnh theo VPS qua `docker stats`).
- Headless tăng nguy cơ Sentinel/Cloudflare detect → toggle headed trong UI (Xvfb sẵn sàng).

### Verify image (không cần combo)

```bash
bash scripts/docker-smoke-test.sh    # 10 check: CLI, Camoufox/GeoIP baked offline, non-root, migrate, compose, xvfb-run
```

Exit 0 = pass, 1 = lỗi image, 2 = skip (docker vắng). AC signup thật: chạy 1 job qua UI/CLI với combo.

## Configuration

### Environment Variables (.env)

**Generated by setup.sh, customize as needed:**

```env
# Browser automation
BROWSER_ENGINE=camoufox              # or 'playwright'
BROWSER_RANDOM_SCREEN=true           # false = fixed viewport (faster, less stealth)

# Runtime paths
RUNTIME_DIR=runtime                  # all profiles, sessions, DB stored here

# Signup job settings
HYBRID_MAX_CONCURRENT=2              # workers (1-10); increase if proxy available
HYBRID_JOB_TIMEOUT=240               # seconds; must be > 180 (OTP timeout)
HYBRID_OUTLOOK_PROXY=                # seed proxy list (comma-separated URLs)

# iCloud HME runner
ICLOUD_RETRY_INTERVAL=900            # seconds between cycles (min 10)
ICLOUD_MAX_ERRORS_PER_CYCLE=0        # error cap per cycle (0 = unlimited)
ICLOUD_API_AUTH_TOKEN=               # Bearer token for /api/icloud/* (if non-loopback)

# Web server
GPT_SIGNUP_WEB_TOKEN=                # override auto-generated token (optional)
```

### Runtime Settings (Web UI)

Access via **Settings** tab after starting web server:

| Setting | Default | Range | Purpose |
|---------|---------|-------|---------|
| `reg.headless` | false | bool | Browser display mode |
| `reg.browser_engine` | camoufox | camoufox, playwright | Browser type |
| `proxy.pool_strategy` | round_robin | round_robin, random | Proxy selection |
| `proxy.max_retries` | 3 | 1-10 | Retries on proxy fail |
| `mail.provider` | outlook | outlook, worker, dongvanfb, gmail | OTP backend |
| `job.timeout` | 240 | 30-600 | Job max duration (sec) |
| `upi.approve_retries` | 10 | 1-50 | UPI confirm retry limit |

**Persist to SQLite:** Settings saved immediately, survive restart.

## Mail Provider Setup

### Option 1: Outlook (Microsoft Graph)

**Requirements:**
- Outlook.com account with app password
- Microsoft Azure app registration (client_id)
- OAuth refresh token (1 per combo)

**How to get:**
1. Create Azure app (portal.azure.com)
2. Add Mail.Read scope
3. Perform OAuth flow manually or use `codex_auth/`
4. Extract refresh_token from response
5. Format: `email|pass|refresh_token|client_id`

### Option 2: Cloudflare Worker (iCloud Relay)

**Requirements:**
- Worker deployed at `https://your-worker.workers.dev/logs`
- Worker logs requests containing OTP codes
- API key for auth

**How to use:**
```bash
.venv/bin/python -m gpt_signup_hybrid signup \
  --email user@icloud.com \
  --mail-provider worker \
  --logs-url https://your-worker.workers.dev/logs \
  --api-key YOUR_API_KEY
```

### Option 3: DongVanFB

**Requirements:**
- tools.dongvanfb.net account credentials
- Same format as Outlook (email|pass|refresh_token|client_id)

**How to use:**
```bash
.venv/bin/python -m gpt_signup_hybrid signup \
  --outlook-combo "email|pass|token|client" \
  --mail-provider dongvanfb
```

### Option 4: Gmail (Advanced API)

**Requirements:**
- checkotpgmail.live API endpoint
- API key

**How to use:**
```bash
.venv/bin/python -m gpt_signup_hybrid signup \
  --email user@gmail.com \
  --mail-provider gmail \
```

## Proxy Pool Setup

### Add Proxies via Web UI

1. Go to **Settings** → **Proxy & Combos**
2. Paste proxy lines (1 per line):
   ```
   http://user:pass@proxy1.com:8080
   https://user:pass@proxy2.com:8080
   host:port:user:pass
   host:port:user-{SID}:pass
   ```
3. Click **Add Proxies**
4. Test: Click **Test Proxies** button

### Proxy Format

**Standard URLs (backward-compatible):**
```
[http|https|socks5]://[user:pass@]host:port
```

**Simple line format (NEW):**
```
host:port
host:port:user:pass
host:port:user-{SID}:pass
```

The `{SID}` placeholder (case-insensitive: `{SID}` or `{sid}`) gets replaced with a random 8-character session ID on each use, enabling unlimited sticky sessions from a single proxy template.

**Examples:**
```
10.0.0.1:8080
proxy.example.com:3128:user:pass
proxy.example.com:3128:user-{SID}:mypass
http://user:pass@proxy.example.com:3128
https://user-{sid}:pass@proxy.example.com:8080
```

### Proxy Configuration (7 Knobs)

All proxy settings accessible via **Settings** tab (persist to SQLite, survives restart):

| Setting | Default | Range | Notes |
|---------|---------|-------|-------|
| `proxy.probe_endpoint` | `https://api64.ipify.org` | HTTP(S) URL | L4 connectivity test endpoint |
| `proxy.probe_timeout` | 6 sec | 3-15 | Timeout per probe attempt |
| `proxy.max_tries` | 5 | 1-20 | Max SID rotations before direct fallback |
| `proxy.sid_len` | 8 | 4-32 | Length of generated session ID |
| `proxy.sid_retry_per_line` | 2 | 0-10 | Retries per proxy line (IP-level fail) |
| `proxy.probe_concurrency` | 4 | 1-10 | Async semaphore bound (requires restart) |

**Environment variables (.env):**
```env
PROXY_PROBE_ENDPOINT=https://api64.ipify.org
PROXY_PROBE_TIMEOUT=6
PROXY_MAX_TRIES=5
PROXY_SID_LEN=8
PROXY_SID_RETRY_PER_LINE=2
PROXY_PROBE_CONCURRENCY=4
```

Priority: Settings store (UI) > `.env` > defaults.

### Proxy Pool Behavior

- **Health-check:** all 4 login flows (UPI, Session, Link, Reg) probe each proxy before use
- **SID rotation:** if IP-level fail (timeout/conn-reset), rotate {SID} placeholder, retry same line
- **Mark dead:** auth fail (407) or host-unresolved → skip entire raw line for session
- **Fallback:** when pool empty or exhausted `max_tries` → DIRECT (no proxy)
- **Concurrency:** up to `probe_concurrency` jobs probe in parallel (prevents thundering-herd)

## LAN Exposure (Non-Loopback)

### Enable Public Access

```bash
.venv/bin/python -m gpt_signup_hybrid web \
  --host 0.0.0.0 \
  --port 8083 \
  --unsafe-expose-network
```

**Output:**
```
WARNING: Binding to 0.0.0.0 (public interface)
AUTH TOKEN: gsh_abc123def456...

Access from remote machine:
  http://192.168.1.100:8083/?token=gsh_abc123def456...
```

### Share Token Securely

1. Copy token from startup output
2. Share via secure channel (Telegram, Signal, encrypted email)
3. User pastes in URL: `http://<your-ip>:8083/?token=<token>`
4. Frontend reads from URL query → localStorage

### Security Considerations

- **Loopback only** (127.0.0.1): recommended for personal use
- **LAN only** (192.168.x.x): acceptable within trusted network
- **Public:** NOT recommended (no HTTPS, no rate limiting)
- **Rotate token:** periodically change `GPT_SIGNUP_WEB_TOKEN` in .env

## Database Backup & Restore

### Backup SQLite DB

```bash
# Simple copy (WAL mode safe)
cp runtime/data.db runtime/data.db.backup

# Or with timestamp
cp runtime/data.db runtime/data.db.backup.$(date +%Y%m%d-%H%M%S)
```

### Restore from Backup

```bash
cp runtime/data.db.backup runtime/data.db
# Restart web server — DB recovered automatically
```

### Export Job History

```bash
sqlite3 runtime/data.db << EOF
.headers on
.mode csv
.output job_history.csv
SELECT * FROM jobs;
.quit
EOF
```

### Import Combo Pool Backup

```bash
sqlite3 runtime/data.db << EOF
-- Clear existing
DELETE FROM outlook_combos;

-- Import from CSV
.mode csv
.import combos.csv outlook_combos
.quit
EOF
```

## Troubleshooting

### Common Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| **Web won't start** | Port 8083 in use | Change: `--port 9000` |
| **Firefox binary missing** | Download failed | Re-run `bash setup.sh` |
| **SQLite locked** | Multiple processes | Kill old web server, restart |
| **OTP timeout** | Mail API slow/down | Increase `HYBRID_JOB_TIMEOUT` |
| **Proxy auth fails** | Bad credentials | Test proxy with curl first |
| **Sentinel PoW fails** | QuickJS subprocess issue | Check Node.js installed, use Python fallback |
| **2FA enroll fails** | Timing issue | Manually enroll TOTP, import secret |
| **Memory leak** | Long-running jobs | Restart web server weekly |

## Security Checklist

### Before Production/LAN Exposure

- [ ] Change default auth token (`GPT_SIGNUP_WEB_TOKEN` in .env)
- [ ] Review proxy credentials (never log passwords)
- [ ] Check `.env` not committed to git (add to `.gitignore`)
- [ ] Set `HYBRID_MAX_CONCURRENT` based on proxy count
- [ ] Enable HTTPS proxy tunnel (if exposing externally)
- [ ] Firewall rules: only trusted IPs
- [ ] Rotate Outlook tokens periodically
- [ ] Backup `runtime/data.db` before major changes
- [ ] Review logs for exposed credentials
- [ ] Disable/hide iCloud HME tab if not needed

### API Token Rotation

1. Generate new token:
   ```bash
   python3 -c "import secrets; print(secrets.token_urlsafe(32))"
   ```
2. Update `.env`: `GPT_SIGNUP_WEB_TOKEN=<new-token>`
3. Restart web server
4. Notify users of new token

## Performance Tuning

### For Throughput (Many Jobs)

```env
HYBRID_MAX_CONCURRENT=5-10       # increase workers
HYBRID_OUTLOOK_PROXY=...         # seed more proxies
ICLOUD_RETRY_INTERVAL=600        # reduce HME cycle time
```

Then monitor:
- CPU usage (should stay <60%)
- Memory usage (watch for leaks, target <500MB)
- Job latency (track P50/P95/P99)

### For Stability (Few Jobs, Long Running)

```env
HYBRID_MAX_CONCURRENT=1-2        # reduce contention
HYBRID_JOB_TIMEOUT=300-600       # increase timeout
```

### Database Optimization

```bash
# Rebuild indexes (monthly)
sqlite3 runtime/data.db "REINDEX;"

# Vacuum (monthly, slower but shrinks file)
sqlite3 runtime/data.db "VACUUM;"

# Check integrity
sqlite3 runtime/data.db "PRAGMA integrity_check;"
```

## Scaling Beyond Local

### NOT Recommended For

- Public internet exposure (no HTTPS, rate limiting)
- Multi-tenant use (shared token, no user isolation)
- High-scale deployment (>500K jobs, single SQLite)

### If Scaling Required (v3.0.0+)

1. **Migrate to PostgreSQL** (shared database)
2. **Add Redis** (session cache, rate limiting)
3. **Deploy behind reverse proxy** (nginx/Caddy with HTTPS)
4. **Add load balancer** (for multiple app instances)
5. **Implement OAuth2** (multi-user support)
6. **Set up monitoring** (Prometheus, Grafana)
